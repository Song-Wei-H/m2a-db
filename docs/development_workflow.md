# Additional Workflow Rules

Before implementation

1. Read PROJECT_CONTEXT.md
2. Read docs/architecture.md
3. Read docs/security_rules.md
4. Read docs/roadmap.md

Before coding

1. Explain implementation plan
2. List files to modify
3. List schema changes
4. Wait for approval

Implementation Rules

* preserve completed stages
* preserve security boundaries
* do not refactor unrelated modules
* implement only approved stage
* update PROJECT_CONTEXT.md after completion
* update roadmap.md when stage status changes
* create migration when schema changes
